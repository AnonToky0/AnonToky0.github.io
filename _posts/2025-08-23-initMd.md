## 1. 标题语法

***

在单词或短语前添加井号`#`。`#`的数量代表了标题的级别。
- 可使用范围为一级标题 到 六级标题
- 在标题和 `#`之间要添加空格
- 可以在文本下方添加任意数量的 == 号来标识一级标题，或者 -- 号来标识二级标题。

## 2. 段落语法

***

使用空白行将文本进行分割来创建段落。
- 不要用空格（spaces）或制表符（ tabs）缩进段落。

## 3. 换行语法

***

在行末添加两个或多个空格， 然后按回车键。
- 也可以使用 HTML的`<br>`标签，这个兼容性也很好，而且能解决空格在编译器中不易被察觉的问题。

## 4. 强调语法

***

### 粗体
在文本的前后添加两个星号`**`或下划线`__`
- 如果要加粗一个单词或短语的中间部分，在要加粗部分的前后添加两个星号`**`

### 斜体
在文本的前后添加一个星号`*`或下划线`_`
- 如果要斜体一个单词或短语的中间部分，在要斜体部分的前后添加两个星号`*`

### 粗斜体
在文本的前后添加三个星号`***`或下划线`___`
- 如果要粗斜体一个单词或短语的中间部分，在要粗斜体部分的前后添加两个星号`***`

## 5. 引用语法

***

要创建块引用，在段落前添加一个 > 符号。

### 多个段落的块引用
块引用可以包含多个段落。为段落之间的空白行添加一个 > 符号。

### 嵌套块引用
块引用可以嵌套。在要嵌套的段落前添加一个 >> 符号。

### 带有其它元素的块引用
块引用可以包含其他 Markdown 格式的元素。并非所有元素都可以使用。

## 6. 列表语法

***

### 有序列表
要创建有序列表，在每个列表项前添加数字并紧跟一个英文句点。
- 数字不必按数学顺序排列，但是列表应当以数字 1 起始。

### 无序列表
要创建无序列表，在每个列表项前面添加破折号 `-`、星号 `*` 或加号`+`
- 缩进一个或多个列表项可创建嵌套列表。 

### 列表嵌套
在保留列表连续性的同时在列表中添加另一种元素，请将该元素缩进四个空格或一个制表符
- 代码块被放在列表中时，请将它们缩进八个空格或两个制表符。


## 7. 代码语法

***

要将单词或短语表示为代码，将其包裹在反引号 `` ` `` 中。

### 转义反引号
如果要表示为代码的单词或短语中包含一个或多个反引号，则可以通过将单词或短语包裹在双反引号中。

### 代码块
创建代码块，请将代码块的每一行缩进至少四个空格或一个制表符。

## 8. 分割线语法

***

在单独一行上使用三个或多个星号 (***)、破折号 (---) 或下划线 (___) ，并且该行不能包含其他内容。
- 为了兼容性，**在分隔线的前后均添加空白行**。

## 9. 链接语法

***

链接文本放在中括号内，链接地址放在后面的括号中，链接Title可选。  

    链接语法： [Markdown语法](https://markdown.com.cn "链接title")。

### 链接title
链接title是当鼠标悬停在链接上时会出现的文字。可选，放在圆括号中链接地址后面，跟链接地址之间以空格分隔。

### 网址和Email地址
使用尖括号包裹可以把URL或者email地址变成可点击的链接。

### 带格式化的链接
在链接语法前后增加星号。 要将链接表示为代码，在方括号中添加反引号。
```
I love supporting the **[EFF](https://eff.org)**.
This is the *[Markdown Guide](https://www.markdownguide.org)*.
See the section on [`code`](#code).
```
效果如下：<br>
I love supporting the **[EFF](https://eff.org)**.<br>
This is the *[Markdown Guide](https://www.markdownguide.org)*.<br>
See the section on [`code`](#code).

### 引用式链接(Reference-style links)
引用样式链接使URL在Markdown中更易于显示和阅读。<br>
它是一种将链接文本和链接地址分开写的链接写法，方便统一管理和复用链接地址。<br>
由两部分组成:
1. 链接文本部分(正文使用)，格式为：<br>
```
[链接文本][引用标识]
```
2. 链接地址定义部分（通常写在文档末尾）
```
[引用标识]: 链接地址 "可选标题"
```  

#### 引用式链接的格式
##### 第一部分格式
尽管不是必需的，可以在第一组和第二组括号之间包含一个空格。  
第二组括号中的标签不区分大小写，可以包含字母，数字，空格或标点符号。
##### 第二部分格式
1. 引用标识，其后紧跟一个冒号和至少一个空格
2. 链接的URL，可以选择将其括在尖括号中。
3. 链接的可选标题，可以将其括在双引号，单引号或括号中。

#### 链接的注意事项
在URL中的空格尽量使用`%20`代替，以保证兼容性。如
```
[link](https://www.example.com/my%20great%20page)
```

## 10. 图片语法
使用感叹号 (!), 然后在方括号增加替代文本，图片链接放在圆括号里，括号里的链接后可以增加一个可选的图片标题文本
```
![图片alt](图片链接 "图片title")
```

## 11. 转义字符语法

***

在字符前面添加反斜杠字符 `\` 
### 特殊字符处理
HTML 文件中，有两个字符 `<` 和 `&`需要特殊处理, `<`符号用于起始标签，`&` 符号则用于标记 HTML 实体。
如果你只是想要使用这些符号，你必须要使用实体的形式，像是 `&lt;` 和 `&amp;`。<br>
Markdown允许直接使用这两个符号，它帮你自动转义字符。  
如果使用 & 符号的作为 HTML 实体的一部分，比如`&nbsp;`, `&copy;`那么它不会被转换，而在其它情况下，它则会被转换成 `&amp;`

## 12. 内嵌 HTML 标签

***

对于 Markdown 涵盖范围之外的标签，都可以直接在文件里面用 HTML 本身。如需使用 HTML，不需要额外标注这是 HTML 或是 Markdown，只需 HTML 标签添加到 Markdown 文本中即可。  
### 行级内联标签
HTML 的行级內联标签如 `<span>`、`<cite>`、`<del>` 不受限制，可以在 Markdown 的段落、列表或是标题里任意使用。
### 区块标签
区块元素──比如 `<div>`、`<table>`、`<pre>`、`<p>` 等标签，必须在前后加上空行，以便于内容区分。而且这些元素的开始与结尾标签，**不可以用 tab 或是空白来缩进**。Markdown 会自动识别这区块元素，避免在区块标签前后加上没有必要的 `<p>` 标签。
- Markdown 语法在 HTML 区块标签中将不会被进行处理。例如，无法在 HTML 区块内使用 Markdown 形式的`*强调*`。
